## 什么是PEP
PEP是 `Python Enhancement Proposal` Python 增强提案的缩写。每篇 PEP 都是一份设计文档，它可能为 Python 社区提供了一些信息，也可能描述了 Python 或其过程和环境的某项新功能特性。如果是功能特性描述，那么 PEP 文档应该提供出一份该特性的简明技术规范及其的基本原理。

我们希望 PEPs 能够成为一种基本制度：可以通过它提出一些重大的新特性，可以通过它收集社区中针对某个问题各种意见，还可以通过它记录一些 Python 中的设计决策。而 PEP 的作者也有责任在社区中建立共识，同时也记录下不同的意见。

PEPs 会以文本文件的形式维护在版本仓库中，所以它们的版本历史也是该功能特性提案的历史记录。
